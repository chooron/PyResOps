# S05 梅台过渡期动态降水位场景

## 场景概述

**场景名称：** 梅汛转台汛过渡期动态降水位调度  
**季节背景：** 梅台过渡期（7月1日～7月15日）  
**核心目标：** 在15天内将库水位从梅汛上限 160.00m 逐步降至台汛限制水位 156.50m，同时应对可能的降雨来水，实现动态滚动调度  
**参考文件依据：** 2.3节汛期划分、2.5节调度权限说明、一、调度原则第（一）汛期洪水调度

---

## 背景说明

根据《2025年度水库控制运用计划》：

- 梅台过渡期：**7月1日～7月15日**（共15天）
- 目标：水位由 **160.00m** 逐步过渡至 **156.50m**
- 实现手段：「通过机组满发和开启溢洪道泄洪等方式」
- 要求：「接省、市气象部门预报有台风影响时，必须及时将库水位降至156.50m」
- 调度权限：过渡期内，若水位仍超 160.00m，调度权在丽水市水利局；降至以下后归电站

本场景核心难点：
1. **不确定性处理**：15天内可能发生梅雨尾声降水，来水量不确定
2. **动态目标切换**：目标水位从 160m 线性降至 156.5m
3. **滚动调度验证**：每天根据实时来水更新计划（reassess_plan）

## 调度规则（动态目标切换）

```python
# 过渡期动态目标水位曲线（线性消落）
# 每天目标水位 = 160 - (160 - 156.5) / 15 * day_index
target_level_schedule = [
    {"day": d, "target_level": round(160.0 - 3.5 * d / 14, 2)}
    for d in range(15)
]
# day 0: 160.0m → day 14: 156.5m

rules = [
    {
        "name": "过渡期积极泄水规则",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "gt", "value": 156.5},
                {"path": "state.level", "op": "lte", "value": 160.0},
                {"path": "directives.transition_mode", "op": "eq", "value": True}
            ]
        },
        "actions": [
            {
                "type": "set_target_outflow",
                "parameters": {
                    "value": 1500.0,      # 机组满发(~600) + 溢洪道少量泄流(~900)
                    "reason": "梅台过渡期主动泄水，按计划降至156.5m"
                }
            }
        ]
    },
    {
        "name": "水位已达台汛限制停止泄洪",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "lte", "value": 156.5}
            ]
        },
        "actions": [
            {
                "type": "set_target_outflow",
                "parameters": {
                    "value": 400.0,       # 仅机组发电
                    "reason": "已降至台汛限制水位，停止额外泄流"
                }
            }
        ]
    },
    {
        "name": "小洪水来临时临时加大泄量",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "gt", "value": 158.0},
                {"path": "forecast.inflow", "op": "gt", "value": 400.0}
            ]
        },
        "actions": [
            {
                "type": "clamp_outflow",
                "parameters": {
                    "min": 1500.0,
                    "max": 4000.0,        # 加大泄量配合洪水通过
                    "reason": "过渡期来水偏大，加大泄量确保水位持续降低"
                }
            }
        ]
    }
]

directives = {
    "transition_mode": True,           # 梅台过渡期模式
    "target_final_level": 156.5,       # 目标台汛限制水位
    "deadline_days": 15,               # 必须在15天内完成
    "target_level_schedule": target_level_schedule  # 逐日目标水位
}
```

## 滚动调度配置

```python
# 使用滚动调度服务，每天更新计划
rolling_config = {
    "horizon_hours": 72,          # 预见期3天
    "step_hours": 24,             # 步长1天
    "update_interval_hours": 24,  # 每天更新一次
    "plan_id": "transition_2025_07"
}

# 滚动调度工具调用序列：
# 1. optimize_flexible_release_plan()  → 生成初始15天计划
# 2. 每天 reassess_plan()              → 根据实测来水更新后续计划
# 3. 若提前完成 replace_working_plan() → 更新为台汛期发电计划
# 4. 15天结束 finalize_plan()          → 固化最终调度记录
```

## 验证要点

| 验证点 | 期望结果 |
|--------|---------|
| 水位目标 | 7月15日末水位 ≤ 156.50m |
| 水位上限 | 全程水位不超过 160.00m |
| 来水反弹 | 场景B下（day 3-5有降水），水位能否仍按期降至156.5m |
| 流量平稳 | 相邻日流量变化 ≤ 500 m³/s |
| 生态约束 | 全程出库 ≥ 50 m³/s |
| 滚动更新 | `reassess_plan` 每天更新后，计划路径合理收敛 |
| 决策透明 | 每步包含目标水位与当前水位的偏差记录 |

## MCP 调用链（滚动调度模式）

```
Day 1（初始计划）：
  1. get_reservoir_snapshot()
     → level=160.0m

  2. optimize_flexible_release_plan(
         snapshot=snapshot,
         forecast=forecast_15days,
         policy_bundle={
             "constraints": constraints,
             "rules": rules,
             "directives": directives
         }
     )
     → 生成15天降水位计划，返回 plan_id

Day 2（滚动更新）：
  3. get_reservoir_snapshot()
     → 获取第2天实测水位（如 159.6m，比计划快/慢？）

  4. reassess_plan(
         plan_id=plan_id,
         snapshot=day2_snapshot,
         updated_forecast=forecast_remaining_14days  # 更新剩余预报
     )
     → 重新评估计划，输出调整建议

  5. replace_working_plan(plan_id, adjusted_plan)
     → 以新计划替换工作计划

Day 15（计划结束）：
  6. get_working_state()
     → 确认当前水位已降至 156.5m

  7. finalize_plan(plan_id)
     → 固化调度记录，转入台汛期常规调度
```

## 场景复杂度分析

本场景是验证 `RollingOpsService` 的核心场景：

| 能力维度 | 验证内容 |
|---------|---------|
| 滚动更新 | `reassess_plan` 能否在新来水情况下合理调整计划 |
| 动态目标 | 逐日目标水位（线性消落曲线）是否能作为约束/目标输入 |
| 不确定性鲁棒 | 来水反弹（场景B）下，调度方案能否自适应 |
| 计划持久化 | `plan_id` 跨多次更新是否保持一致性 |
| 工作状态管理 | `get_working_state` 能否反映最新计划执行情况 |

## 预期调度路径

```
场景A（平稳消退）：
  Day 1-3：出库 1500 m³/s，来水 200→170→140 m³/s，水位降至 159.5m
  Day 4-8：出库 1200 m³/s，水位降至 158.2m
  Day 9-14：出库 1000 m³/s，水位降至 156.6m
  Day 15：水位达到 156.5m，计划完成 ✓

场景B（来水反弹）：
  Day 1-2：出库 1500 m³/s，水位降至 159.7m
  Day 3-5：来水 300→450→350，水位反弹至 160.0m（触发上限约束）
  Day 5 reassess_plan：识别进度滞后，建议加大出库至 2500 m³/s
  Day 6-10：出库 2500 m³/s，水位迅速降至 158m
  Day 11-15：出库 1500 m³/s，水位降至 156.5m ✓（险险达标）
```

## 扩展方向

- 接入台风路径预报：若预报台风登陆时间提前，缩短降水位窗口（如从15天压缩到10天）
- 与区间洪水联合：过渡期如有区间洪水，同步启用错峰调度
- 多电站协调：与上游紧水滩电站协调，通过梯级联调加速腾库
