# S04 枯水期发电优化场景

## 场景概述

**场景名称：** 枯水期兴利发电优化调度  
**季节背景：** 枯水期（10月～次年3月）  
**核心目标：** 在满足生态流量的前提下，优化水库消落过程，最大化发电效益，合理安排月度发电计划  
**参考文件依据：** 3.4节2025年调度计划表、第（三）兴利调度原则、第（四）生态流量控制

---

## 背景说明

根据《2025年度水库控制运用计划》3.4节分月调度计划，枯水期（10～12月）计划数据：

| 月份 | 来水量(亿m³) | 发电量(万kWh) | 耗水率(m³/kWh) | 月初水位(m) | 月末水位(m) |
|------|------------|-------------|--------------|-----------|-----------|
| 10月 | 1.20 | 5500 | 3.40 | 150.35 | 149.18 |
| 11月 | 1.40 | 5500 | 3.42 | 149.18 | 148.32 |
| 12月 | 1.10 | 4500 | 3.43 | 148.32 | 147.52 |

年均耗水率约 3.40 m³/kWh，设计保证出力 87.75 MW（P=90%）。

## 调度模块配置

```python
# 枯水期主要依赖存储驱动 + 等出力调度
dispatch_modules = [
    {
        "type": "storage_driven",
        "parameters": {
            "target_level_curve": [
                {"month": 10, "target": 149.18},
                {"month": 11, "target": 148.32},
                {"month": 12, "target": 147.52}
            ],
            "description": "按计划水位消落曲线"
        }
    },
    {
        "type": "constant_release",
        "parameters": {
            "target_power": 87.75,    # 保证出力运行（低来水阶段）
            "min_outflow": 50.0,
            "description": "枯水期保证出力运行"
        }
    }
]

# 规则：来水很小时优先保生态流量
rules = [
    {
        "name": "极枯来水保生态\",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.inflow", "op": "lt", "value": 50.0}
            ]
        },
        "actions": [
            {
                "type": "set_target_outflow",
                "parameters": {
                    "value": 50.0,    # 最低生态流量
                    "reason": "来水极枯，仅保障生态流量，库水位维持"
                }
            }
        ]
    },
    {
        "name": "水位接近死水位时限制出库",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "lt", "value": 125.0}
            ]
        },
        "actions": [
            {
                "type": "clamp_outflow",
                "parameters": {
                    "min": 50.0,
                    "max": 100.0,      # 接近死水位时严格控制出库
                    "reason": "水位接近死水位，减少出库保库容"
                }
            }
        ]
    }
]
```

## 验证要点

| 验证点 | 期望结果 |
|--------|---------|
| 生态流量 | 全程出库 ≥ 50 m³/s，无生态约束违反 |
| 发电量达标 | 3个月发电量 ≥ 15500万kWh（计划值） |
| 水位消落合理 | 12月末水位在 147.52m 附近（误差 ≤0.5m） |
| 无弃水 | 枯水期 `spillage = 0`（来水小，无需弃水） |
| 耗水率合理 | 平均耗水率约 3.40～3.43 m³/kWh |
| 死水位安全 | 全程水位 > 120m |

## MCP 调用链

```
1. get_reservoir_snapshot()
   → 获取10月初水库状态

2. generate_dispatch_program(
       snapshot,
       forecast=daily_forecast_90days,
       modules=["storage_driven", "constant_release"]
   )
   → 生成90天兴利调度方案

3. simulate_program(
       program,
       policy_bundle={
           "constraints": constraints,
           "rules": rules,
           "objectives": objectives
       }
   )
   → 90天仿真，输出逐日水位、出库、发电量

4. evaluate_program(sim_result)
   → power_score（发电得分）
   → ecology_score（生态得分）
   → compliance_score（合规得分，水位是否按计划消落）

5. compare_programs([plan_A, plan_B])
   → 对比不同出力目标下的发电量与水位消落差异

6. explain_program(sim_result)
   → 说明极枯来水时段的决策逻辑
```
