# S03 极端洪水应急调度场景

## 场景概述

**场景名称：** 超标准洪水应急调度（500年一遇）  
**季节背景：** 台汛期  
**核心目标：** 应对超过防洪高水位的极端洪水，保障大坝安全，同时尽量减少下游损失  
**参考文件依据：** 2.5节防洪调度原则第④⑤⑥条，4.3节应急预案相关规定

---

## 背景说明

根据调度规则：

| 水位区间 | 调度措施 |
|---------|---------|
| ≤161.5m | 补偿凑泄，控制下游≤14000 m³/s |
| 161.5m < Z ≤ 161.7m | 按 6000 m³/s 控泄（缓冲区间） |
| Z > 161.7m | 溢洪道全开，机组参与泄洪 |
| Z > 165.27m（P=0.2%） | 溢洪道全开 + 开启泄洪洞，停止发电 |

本场景模拟 **P=0.5%（500年一遇）量级洪水**，入库洪峰约 17500 m³/s，水位将超过设计洪水位（165.87m）。

## 验证要点

| 验证点 | 期望结果 |
|--------|---------| 
| 大坝安全 | 全程水位不超过校核洪水位 169.15m |
| 阶梯式决策切换 | 4个水位区间对应4种不同调度策略，规则切换准确 |
| 泄洪洞启用触发 | 水位超过 165.27m 时，`emit_event` 或 `outflow` 增至泄洪洞满容量 |
| 发电停止逻辑 | 水位 > 165.27m 时机组流量 = 0（记录在决策追踪中） |
| 最大泄量约束 | 溢洪道单独泄量 ≤ 14335 m³/s |
| 洪峰消减 | 洪峰拦蓄率有效（100年一遇洪水成果：拦蓄率约30%） |
| 全程轨迹完整 | 72步决策轨迹均有记录，包括规则切换点 |

## MCP 调用链

```
1. get_reservoir_snapshot()
   → 确认起调水位 156.5m（已腾出防洪库容）

2. generate_dispatch_program(
       snapshot, forecast_72steps,
       modules=["flexible_release"]
   )
   → 生成应急调度方案

3. simulate_program(
       program,
       policy_bundle={
           "constraints": constraints,
           "rules": rules,          # 4条分级规则
           "objectives": objectives
       }
   )
   → 模拟洪水过程，关键输出：
     - max_level（最高水位）
     - decision_trace_steps（规则切换记录）
     - constraint_violations（是否违反校核洪水位）

4. evaluate_program(sim_result)
   → flood_control_score（防洪效益评分）

5. 若水位接近校核洪水位，触发 reassess_plan()
   → 动态调整出库方案保障大坝安全
```

## 关键决策节点

```
Step 10（入库洪峰 17500 m³/s）：
  → 水位约 164.8m（超过防洪高水位，低于设计洪水位）
  → 规则：「超过161.7m溢洪道全开」
  → 出库：溢洪道6孔全开约11000m³/s + 机组400m³/s = ~11400 m³/s
  → 洪峰被削减：(17500-11400)/17500 ≈ 35%

Step 15（水位接近设计洪水位）：
  → 水位约 165.4m（超过 165.27m P=0.2% 阈值）
  → 规则：「超过500年一遇水位启用泄洪洞」
  → 出库：溢洪道 11085 m³/s + 泄洪洞约1699 m³/s = 12784 m³/s，发电停止
  → 决策轨迹记录「发电暂停」事件

Step 30（退水阶段，水位降至160m以下）：
  → 规则切换回「正常调度」
  → 恢复机组发电流量
```
