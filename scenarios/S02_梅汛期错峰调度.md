# S02 梅汛期错峰调度场景

## 场景概述

**场景名称：** 梅汛期下游青田县城错峰调度  
**季节背景：** 梅汛期（4月15日～6月30日）  
**核心目标：** 在洪水来临时，控制滩坑水库下泄流量，使鹤城站流量不超过 14000 m³/s（20年一遇），保障青田县城防洪安全  
**参考文件依据：** 2.5节防洪调度原则第②③条、一、调度原则第（一）汛期洪水调度第2条

---

## 背景说明

根据《2025年度水库控制运用计划》：

- 库水位在 160.00m（梅汛期限制水位）至 161.50m（防洪高水位）之间时，实行**补偿凑泄**方式调度
- 下泄控制公式：`Q_泄 = Q_安全 - DQ_1 - Q_机`
  - `Q_安全 = 14000 m³/s`（鹤城站安全泄量）
  - `DQ_1`：考虑洪水传播时间（5.0h）后的区间流量
  - `Q_机`：机组发电流量（约 400 m³/s）
- 若区间洪水将达到或超过 14000 m³/s，水库关闸，为青田减轻压力
- 当库水位超过 161.50m 时，逐渐开启溢洪道直至全开

## 调度规则

```python
rules = [
    {
        "name": "梅汛期正常水位错峰调度",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "gt",  "value": 160.0},
                {"path": "state.level", "op": "lte", "value": 161.5}
            ]
        },
        "actions": [
            {
                "type": "clamp_outflow",
                "parameters": {
                    # 下泄 = 安全泄量 - 区间流量 - 发电流量
                    # 粗估：14000 - 5000(区间洪峰) - 400(发电) = 8600 m³/s
                    "min": 400.0,
                    "max": 8600.0,
                    "reason": "补偿凑泄：控制鹤城站不超过14000m³/s"
                }
            }
        ]
    },
    {
        "name": "水位超过防洪高水位开闸泄洪",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "gt",  "value": 161.5},
                {"path": "state.level", "op": "lte", "value": 161.7}
            ]
        },
        "actions": [
            {
                "type": "set_target_outflow",
                "parameters": {
                    "value": 6000.0,
                    "reason": "水位161.5m～161.7m区间，按6000m³/s控泄"
                }
            }
        ]
    },
    {
        "name": "水位超过161.7m溢洪道全开",
        "metadata": {"rule_type": "expression"},
        "conditions": {
            "all": [
                {"path": "state.level", "op": "gt", "value": 161.7}

                "type": "set_target_outflow",

                    "value": 400.0,

            }
        ]
    }
]
```

## 验证要点

| 验证点 | 期望结果 |
|--------|---------|
| 错峰效果 | 任意时步：`outflow + interval_flow ≤ 14000 m³/s` |
| 水位上限 | 库水位不超过 161.5m（防洪高水位） |
| 关闸逻辑 | 区间洪水预报超限时出库回落至 ≤400 m³/s |
| 洪峰消减 | `(peak_inflow - max_outflow) / peak_inflow ≥ 30%` |
| 拦蓄效果 | 洪水拦蓄率与 2024年实测（约 30%）量级相当 |
| 约束违反 | 下游安全泄量约束 `constraint_violations = []` |

## MCP 调用链

```
1. get_reservoir_snapshot()
   → 获取当前水库快照

2. generate_dispatch_program(
       snapshot, forecast,
       modules=["flexible_release", "constant_release"]
   )
   → 生成错峰调度方案

3. simulate_program(
       program,
       policy_bundle={
           "constraints": constraints,
           "rules": rules,
           "objectives": objectives,
           "directives": {
               "downstream_interval_flows": downstream_interval_flows,
               "propagation_hours": 5.0,
               "downstream_safe_flow": 14000
           }
       }
   )
   → 仿真执行，生成决策轨迹

4. evaluate_program(sim_result)
   → 防洪得分、发电得分、约束违反数

5. explain_program(sim_result)
   → 关键决策点解释（关闸、开闸时机及原因）
```

## 典型决策路径示意

```
Step 5（洪峰时段）：
  入库: 6000 m³/s
  区间预报: 6000 m³/s（传播5h后到鹤城）
  → 规则命中「梅汛期正常水位错峰调度」
  → 目标出库 = 14000 - 6000 - 400 = 7600 m³/s
  → 约束检查：水位 161.2m ≤ 161.5m ✓，下游 7600+6000=13600 ≤ 14000 ✓
  → 最终出库: 7600 m³/s

Step 6（洪峰过后）：
  入库: 5800 m³/s
  库水位升至 161.4m，区间流量下降至 5500 m³/s
  → 出库调整为 8100 m³/s，鹤城总量 13600 m³/s
```
