# S02 动态执行流程说明

本文档描述的是当前代码里的实际执行方式，对应实现文件主要是 `experiments/dynamic_experiment.py`、`experiments/evaluation_metrics.py` 和 `experiments/paper_experiment_runner.py`。

## 场景定位

`S02` 表示梅汛期错峰调度场景。

当前动态实验里，这个场景的初始状态是：

- 当前水位 `159.8 m`
- 初始库容 `35.09 亿m3`
- 当前入流 `2800 m3/s`
- 预报入流 `3380 m3/s`
- 时步 `3 h`
- 总时长 `48 h`

## 指令更新方式

`S02` 的执行方式和 `S01` 一样，都是阶段式动态执行：

1. 每个阶段只给 LLM 当前状态和当前新增指令。
2. LLM 返回一个当前阶段的单值出库流量。
3. 系统用这个出库流量把状态推进到下一次触发时刻。
4. 下一阶段再基于更新后的真实状态继续调度。

所以，`S02` 不是一次性生成 48 小时完整曲线，而是“阶段决策 -> 状态推进 -> 再决策”的循环。

## LLM 在每阶段看到的内容

每次触发前，系统都会用 `_apply_trigger()` 更新：

- `current_level`
- `initial_storage`
- `initial_inflow`
- `description`

然后再调用 `run_scenario()`。

这意味着 `T1` 看到的是 `T0` 推演后的状态，`T2` 看到的是 `T1` 推演后的状态，依次向后衔接。

## S02 的时间轴

### T0

- 触发时刻：`0 h`
- LLM 看到的状态：初始状态
- 指令文本：控制鹤城站流量不超过 `14000 m3/s`
- 推进窗口：`6 h`
- 达标判断：`llm_outflow <= 14000`

这里有一个必须说明的实现细节：

虽然场景语义说的是“控制鹤城站流量”，但当前代码里的 `flow_limit` 判定并没有把区间流量传播算进去，而是直接检查 LLM 给出的出库流量本身是否不超过上限。

也就是说，当前自动评估是“出库上限判定”，不是“鹤城站真实断面流量判定”。

### T1

- 触发时刻：`6 h`
- LLM 看到的状态：`T0` 推进后的状态
- 指令文本：上限从 `14000` 收紧到 `12000 m3/s`
- 推进窗口：`6 h`
- 达标判断：`llm_outflow <= 12000`

这一阶段的动态性体现为：同一个洪水过程里，控制上限突然变严，LLM 必须在已演变的状态基础上重新收缩出库。

### T2

- 触发时刻：`12 h`
- LLM 看到的状态：`T1` 结束后的状态
- 指令文本：关闸，为青田减压，出库不超过 `400 m3/s`
- 推进窗口：`12 h`
- 达标判断：`llm_outflow <= 400`

这一步是 `S02` 最明显的指令切换点。

系统要求的不再是“较大流量错峰”，而是直接切到“近似关闸”的策略。于是这个场景的执行逻辑会在中途出现非常大的调度风格变化。

### T3

- 触发时刻：`24 h`
- LLM 看到的状态：`T2` 推进后的状态
- 指令文本：逐步恢复出库，但水位不超过 `161.5 m`
- 推进窗口：剩余 `24 h`
- 达标判断：推进后的 `state_after_sim["level"] <= 161.5`

这一步和前几步不同，它不再直接卡出库值，而是检查本阶段推进后的末水位是否超过上限。

因此，`T3` 是一个“状态结果导向”的检查，而不是“出库值导向”的检查。

## 这个场景里状态怎么往后传

每个阶段结束后，系统都用 `_advance_state()` 做水量平衡推进：

1. 入流按场景中的常数 `3380 m3/s` 处理。
2. 出流按当前阶段 LLM 给出的常数流量处理。
3. 按 `3 h` 时步推进库容和水位。

因此，`S02` 的下一阶段状态来源于上一阶段真实推演结果，而不是人工手动设定的固定检查点。

## 这个场景的动态含义

`S02` 的动态执行过程可以理解为：

1. 先做常规错峰。
2. 中途上限收紧。
3. 再进一步切到近似关闸。
4. 最后再在安全水位约束下恢复出库。

所以它本质上是在测试：同一场洪水过程中，LLM 能否跟随调度目标的连续变化，逐阶段调整策略。

## 当前实现里的一个重要限制

这个场景在文字上强调的是“鹤城站控制断面”，但当前程序没有在 `evaluate_instruction_compliance()` 中显式计算断面传播后的站点流量。

因此，当前代码版本里：

- `T0/T1/T2` 的达标判断更接近“出库上限是否服从指令”
- 还不是“下游传播后断面是否严格达标”

如果后续要进一步增强语义一致性，这里是一个明显的可继续细化点。
